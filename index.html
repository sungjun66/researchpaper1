PiLS of Embedded Hybrid MPC for Fighter Aircraft Attack Angle Control Using QP-based Approximation of MINLP

Sungjun Park1, JiWon Kim1, Kwangki Kim1,2,*,
1Department of Electrical and Computer Engineering, Inha University, Republic of Korea 2Department of Electrical and Electronic Engineering, Inha University, Republic of Korea
Paper

Abstract

The control of an aircraft’s attack angle is critical for flight safety. This study presents a control framework that accounts for nonlinear dynamics and discrete input constraints while remaining feasible for real-time implementation.

Focusing on the F-8 Crusader aircraft, the original Mixed-Integer Nonlinear Programming (MINLP) problem is relaxed into a continuous Quadratic Program (QP) and addressed using a Linear Parameter-Varying MPC (LPV-MPC) structure, with discrete constraints approximated via Rule-Based Control (RBC). The framework is validated through Processorin- the-Loop (PiL) simulation on a Teensy 4.1 microcontroller.

Both strategies stabilized the attack angle to 0 rad for prediction horizons of 4 and 8 with a 0.05 s sampling time, and the LPV-MPC+RBC method met the 50 ms real-time control requirement on the embedded platform.

Control Architecture

This figure illustrates the closed-loop control scheme where the LPV-MPC block iteratively performs linearization, discretization, and QP formulation to compute a relaxed continuous control input. This intermediate value is processed by the Rule-Based Control (RBC) logic, which quantizes the signal into specific discrete values (±0.05236 rad) based on its sign. The resulting discrete input is then applied to the nonlinear F-8 aircraft model to update the system states for the next iteration.

Verification Environment

This figure depicts the Processor-in-the-Loop Simulation (PiLS) verification environment, establishing real-time communication between the F-8 Crusader plant model in Simulink and the control algorithm deployed on a Teensy 4.1 microcontroller. The Simulink environment transmits the current state variables to the hardware, while the Teensy board executes the embedded LPV-MPC using the OSQP solver. The computed optimal control input is then sent back to the Simulink plant model to validate real-time feasibility and performance.
